# Auto Radar Global v2 - Roadmap

## Phase Planning

### Phase 0 - Foundation

Goal: establish the core decision framework and dashboard prototype.

Status: completed

Delivered:

* Theme Tree Engine
* Lifecycle Engine
* Theme Score Framework
* Stage Transition Engine
* Risk Engine
* Confidence Framework
* Daily Brief Engine
* Dashboard Prototype
* Constitution
* Team Operating Model
* Role Matrix

### Phase 1 - Data Pipeline and Project Management Core

Goal: replace static dashboard data with structured system data and create durable project memory.

Status: in progress

Scope:

* Daily Brief JSON source
* Data Pipeline v1
* Project Management Core
* Knowledge Core structure
* Changelog discipline
* Roadmap discipline

### Phase 2 - Persistence and Automation

Goal: make Auto Radar store decisions, outcomes, and lessons automatically.

Status: planned

Scope:

* PostgreSQL integration
* Decision Log storage
* Outcome Tracker
* Scheduler validation
* Historical daily brief archive

### Phase 3 - Notification and Review Loop

Goal: deliver actionable daily outputs and collect feedback for learning.

Status: planned

Scope:

* Telegram Bot
* Daily Brief push
* Risk alerts
* Post-decision review workflow
* Root Cause Analysis workflow

### Phase 4 - Adaptive Decision Intelligence

Goal: accumulate market knowledge and improve decision quality through evidence, outcomes, and lessons.

Status: planned

Scope:

* Event Library
* Playbook Library
* Lesson Library
* Hypothesis Library
* Pattern Evolution tracking
* Market Learning reports
* Decision Lab validation

## Long-Term Goals

* Build an Adaptive Decision Intelligence System, not a one-time stock prediction tool.
* Keep every decision traceable to evidence.
* Keep every failed decision reviewable through Root Cause Analysis.
* Convert repeated lessons into playbooks only after validation.
* Prevent unverified hypotheses from entering the formal model.
* Preserve role boundaries: research, architecture, engineering, and final decision remain separate.
