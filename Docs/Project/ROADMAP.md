# Auto Radar Product Roadmap

Status: Active
Authority: Product Roadmap
Last Updated: 2026-06-29

Project state is governed by `PROJECT_STATUS.md`.

## Stage 1 — Research

Status: **Complete / Frozen**

Delivered:

- Market Evolution, Narrative Intelligence, Reality, and Market Flow frameworks.
- Evidence, Case Validation, Learning, MarketMind, and Decision Journal specifications.
- Architecture Constitution, Entity Map, Runtime Authority Matrix, and Research Phase closure.

Boundary:

- No new G-series research task without Architecture Review approval.
- Research assets do not authorize Production behavior.

## Stage 2 — Repository

Status: **Complete**

Delivered:

- Repository entities and indexes for Evidence, Case, Pattern, Experience, Graph, Playbook, Prediction, Outcome, Evaluation, and Review.
- Cross-reference and fail-fast validators.
- Global Repository Validator.
- Evidence-to-Pattern Sandbox knowledge lifecycle.

Boundary:

- Repository completeness is not Runtime authorization.

## Stage 3 — Dashboard

Status: **North Star v1 Implemented / Awaiting Acceptance**

Delivered:

- North Star Hero and Captain route.
- Top3 opportunity panel and no-sail risk boundary.
- Ocean Map, current, tide, regime, and health views.
- Smart Money, density, energy, and velocity views.
- Repository, learning, evolution, and review views.
- Market Story and explainability drawer.
- Responsive desktop and mobile layout.
- Full Traditional Chinese user interface.

Next improvement candidates require separate approved tasks:

- Real approved daily strategy input.
- Approved regime records.
- Validated Opportunity Score records.
- Validated capital-flow graph data.

## Stage 4 — Engine

Status: **Planned / Production Blocked**

Potential scope:

- Bounded Shadow Runtime design.
- Engine contracts, monitoring, rollback, and audit.
- Approved Decision Lab integrations.

Entry gate:

- Architecture specification.
- Validation plan.
- Runtime Authority Matrix permission.

## Stage 5 — Automation

Status: **Planned**

Potential scope:

- Scheduled Repository projections.
- Daily data-package generation.
- Review queue assistance.
- Notifications after provider and failure-mode approval.

Entry gate:

- No external API or automatic write without separate approval.

## Stage 6 — Beta

Status: **Planned**

Definition:

- Daily read-only use with real approved records.
- Stable data refresh.
- User feedback and issue tracking.
- Shadow behavior only where explicitly authorized.

## Stage 7 — v1.0

Status: **Planned**

Definition:

- Daily workflow is reliable.
- Evidence and decisions are traceable.
- Review and learning loops are operational.
- Security, monitoring, rollback, and ownership are documented.
- Product Owner accepts release risk.

## Roadmap Rules

- Stages cannot be skipped.
- `Engineering Ready` is not `Production Ready`.
- A planned item is not an approved task.
- Only `NEXT_TASK.md` may name the active and next task.
