## Declaration

Auto Radar Global v2 Research Phase v1.0 is complete for architecture packaging and is frozen under `Docs/ARCHITECTURE_FREEZE_POLICY.md`.

This declaration closes the Research Phase baseline. It does not assert that every hypothesis is validated and does not authorize Production behavior.

## Completed Modules

The v1.0 baseline includes:

* Project Management Core
* Team Operating System
* Repository Governance
* Market Evolution Framework
* Narrative Intelligence Framework

* Market Flow Framework
* Evidence Layer

* Learning Taxonomy and Learning Score Candidate
* Experience Engine Repository skeleton
* Outcome Repository
* Outcome Evaluation Repository
* Architecture Constitution, Entity Map, Runtime Authority Matrix, and Project Blueprint

## Freeze Scope

The freeze covers:

* architecture hierarchy;
* canonical entity flow;
* role and authority boundaries;
* repository governance;
* candidate versus Production boundaries;
* Runtime deny-by-default policy;
* Research Phase v1.0 documents, schemas, and templates as a baseline.

## What Is Frozen

Without a separately approved change:

* no new Engine may be added;
* no Entity flow may be reordered;
* no role may gain new authority;
* no candidate may become Production;
* no Runtime permission may be enabled;
* no Theme Score, Stage Score, Decision Score, weight, or trading logic may change;
* no Constitution boundary may be bypassed.

## What Remains Allowed

Under the Freeze Policy, the following remain allowed:

* corrections and broken-link repairs;
* Registry and Architecture Debt maintenance;
* evidence and Case collection;
* Outcome, Evaluation, Review, and Learning records;
* schema and template candidates approved by Architecture Review;
* Shadow planning and validation design;
* archive and deprecation metadata;
* governance updates that do not change Production behavior.

## Next Phase: Engineering Layer / Shadow Runtime

The next phase may prepare bounded Engineering and Shadow Runtime work.

All next-phase work must:

* use a dedicated Task ID;
* follow Research -> Architecture -> Development -> Validation;
* preserve deny-by-default Runtime authority;
* state model and Runtime impact;
* avoid Production Rules unless separately approved;
* produce tests, monitoring, and rollback requirements before any behavior change.

## Definition of Done for Leaving Research Phase

Research Phase v1.0 may be treated as closed when:

- [x] Architecture Constitution exists.
- [x] Architecture Portal indexes canonical governance.
- [x] Entity flow and ownership are defined.
- [x] Runtime authority is deny-by-default and documented.
- [x] Repository structure is standardized.
- [x] Project Blueprint supports onboarding.
- [x] Freeze scope and allowed changes are explicit.
- [x] No Runtime, Pipeline, Scoring, Dashboard, Strategy, or Production Rule was changed by the closure package.
