# Playbooks

Purpose:

Store validated repeatable market playbooks.

A playbook is a market pattern that has enough evidence, validation, and review to be reused as structured decision knowledge.

This folder is used for:

* validated theme behavior patterns
* repeatable event reaction patterns
* risk-control playbooks
* stage transition playbooks
* decision review playbooks

Each playbook should preserve:

* playbook name
* market condition
* trigger conditions
* confirmation signals
* invalidation signals
* risk notes
* historical examples
* validation status

Rules:

* Unvalidated ideas belong in `Knowledge/Hypothesis/`, not here.
* Playbooks must not override Constitution rules.
* Stage 6 remains non-tradable.
* Playbooks must include invalidation and risk controls.
